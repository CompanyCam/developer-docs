    CompanyCam API

    CompanyCam helps companies manage field work with smart photos.

    If anything is missing or seems incorrect, please check the GitHub issues for existing known issues or create a new issue.

    Authentication

    To authorize, use this code:

    # With shell, you can just pass the correct header with each request
curl "api_endpoint_here"
  -H "Authorization: Bearer <ACCESS_TOKEN>"
  -H "X-CompanyCam-User: <USER_EMAIL>"

    Make sure to replace <ACCESS_TOKEN> with your API key.

    CompanyCam expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: Bearer <ACCESS_TOKEN>

    When possible you should set the user’s email address performing the action in your app in a header that looks like the following: X-CompanyCam-User: <USER_EMAIL>

    If the email address is not provided it will use the email address of the user that configured the integration.


    You must replace <ACCESS_TOKEN> with your personal API key.
    You should replace <USER_EMAIL> with the user’s email address.

    Getting Started

    Getting an Access Token

    The user can generate a new access token in CompanyCam as long as they have sufficient privileges. Users can generate as many access tokens as they wish. The generated access token will be use in the Authorization header

    Making an API call

    curl -X GET -H "Content-type: application/json" -H "Authorization: Bearer <ACCESS_TOKEN>" -H "X-CompanyCam-User: shawn@psych.com"
https://api.companycam.com/v2/

    The CompanyCam API is entirely JSON based. In order to make an authenticated call to the API, you must include your access token with the call. CompanyCam uses a BEARER token that is passed along in an Authorization header. You should whenever possible set the X-CompanyCam-User HTTP header to the email address of the user performing the action. This email must match the user’s email address in CompanyCam. If the X-CompanyCam-User header is not provided it will fallback to using the user that setup the integration.

    Pagination

    The default limit is 50 items.

    curl -X GET -H "Content-type: application/json" -H "Authorization: Bearer <ACCESS_TOKEN>"
https://api.companycam.com/v2/users?per_page=50&page=1

    The CompanyCam API enables pagination by allowing users to include ‘per_page’ and ‘page’ parameters on GET requests to index pages.

    Note about soft-deletion

    CompanyCam uses soft-deletion for most of its records. In the API, a soft-deleted object will not be returned.

    Permissions Enum

    Title Value Description
    Modify Users 1 Can create and update users
    Delete Users 2 Can delete users
    Modify Groups 3 Can create and update groups
    Delete Groups 4 Can delete groups
    Manage Billing 5 Can view and update the company’s billing information
    Access Reports 6 Can create, update, and delete reports
    Delete Photos 7 Can delete photos
    Move Photos 8 Can move photos between projects
    Share Photos 9 Can share photos by creating things like galleries or sharing timelines
    View Users Photos 10 Can view a users photos if they are part of a project the user has participated in
    View Groups Photos 11 Can view photos of users that are in groups user is a member of
    Access Mobile App 12 Has access to the mobile app
    Modify Project 13 Can create and update projects
    Delete Projects 14 Can delete projects
    Browse User Photos 15 Can view and filter phots by a specific user

    Errors

    The CompanyCam API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request is invalid
    401 Unauthorized – User needs to authenticate or authentication failed
    402 Payment Required – Your subscription has expired
    403 Forbidden – The user doesn’t have privilege to access the resource
    404 Not Found – The specified resource could not be found
    409 Conflict – The entity is not unique
    422 Unprocessable Entity – There was an issue persisting your request due to invalid data
    500 Internal Server Error – We had a problem with our server

    Attribute Type Description Readonly
    id String A unique identifier for the Project. true

    public Boolean Indicates whether the Project’s timeline and other public features are enabled.
    geofence Array[Coordinate] The Project’s geofence coordinates that form an enclosing perimeter.
    created_at Integer Timestamp when the Project was created true
    updated_at Integer Timestamp when the Project was last updated true

    Photo

    This is an object representing a photo of a project. Projects can have multiple photos but a photo object belongs to only one Project.

    The Photo object

    Attribute Type Description Readonly
    id String A unique identifier for the Photo. true
    company_id String A unique identifier for the Company the Photo belongs to. true
    creator_id String The id of the User that took the Photo. true
    creator_type String The type of entity that created this photo. true
    creator_name String The display name of the entity that created the photo. true
    project_id String The id of the Project the Photo was taken at. true
    status String Indicates the status of the Photo. It will be active or deleted true
    coordinates Coordinate The latitude and longitude of the Project.
    uris Array[ImageURI] A list of images for the different variants of the Project’s feature image. Will have entries for original, web, and thumbnail. true
    hash String The MD5 hash of the images contents. true
    internal Boolean Indicates whether the Photo’s is for internal use only and should not be used in marketing or other public materials.
    captured_at Integer Timestamp when the Photo was taken by the User. true
    created_at Integer Timestamp when the Photo was created on the server. This may differ from the captured_at field. true
    updated_at Integer Timestamp when the Photo was last update. true
    photo_url String The link to the photo in the webapp. true

    Types

    The Coordinate object

    Attribute Type Description
    lat Float The latitude of the coordinate
    lon Float The longitude of the coordinate

    The Address object

    Attribute Type Description
    street_address_1 String The first line of the street address. Normally this is the building or house number along with the street name.
    street_address_2 String The second line of the street address. Normally this is the suite or apartment number.
    city String The city in which the item is located
    state String The two character state identifier in which the item is located
    postal_code String The postal code in which the item is located
    country String The short country identifier in which the item is located

    The Role object

    Attribute Type Description
    id String The id of the Role
    name String The name of the Role
    permissions Array[Permission] A list of the permissions belonging to the Role

    The Permission object

    Attribute Type Description
    title String The title of the Permission
    value Integer The value of the Permission that corresponds to the Permission enum

    The ImageURI object

    Attribute Type Description
    type String The type of the URI, for example original or thumbnail
    uri String The full URI of the photo
